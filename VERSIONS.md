# Document versions

_Generated by `build/pdf/check_meta.py` from `build/pdf/versions.json`, which is the
single source of truth for every document's version and date. Do not edit this file
by hand._

**How to update.** When a document changes substantively, edit its line in
`versions.json` — bump the version and set `updated` to today's `as_of` — then
re-render and run `python build/pdf/check_meta.py`. Each PDF's printed stamp is
generated from `versions.json`, so it cannot drift.

**Versioning.** `v0.x` = pre-launch drafts (bump the minor on each substantive
change). `v1.0` = the version published on **2026-07-02**. `v1.1`+ / `v2.0` =
post-launch amendments. A document's date is the day of its last substantive change;
typos and formatting-only edits do not bump the version.
